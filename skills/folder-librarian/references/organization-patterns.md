# Organization Patterns

Pick the pattern based on how the user will retrieve files later. Do not force one universal method.

## Action-Based

Use for active work. Organize around outcomes and next use:

```text
00 Start Here
01 Active Projects
02 Working Docs
03 References
04 Deliverables
90 Archive
```

Good when current status matters more than topic purity.

## Topic Or Library-Based

Use for durable reference archives, training material, playbooks, research, and professional libraries:

```text
00 Start Here - Index
01 Core Frameworks
02 Topic Area A
03 Topic Area B
04 Case Studies
05 Templates And Tools
90 Source Bundles
98 Needs Review
```

Good when the folder should answer future questions.

## Time-Based

Use for admin records, exports, reporting, events, photos, and periodic work:

```text
2026
  2026-01
  2026-02
2025
```

Use dates as `YYYY-MM-DD` when naming files so sorting works naturally.

## Source Or Origin-Based

Use when provenance matters: exports from old machines, client-provided files, vendor reports, source datasets, email attachments.

```text
01 From Client
02 From Vendor
03 Internal Working
04 Derived Outputs
90 Original Exports
```

Keep original bundles intact when the source itself is meaningful.

## Audience Or Client-Based

Use when handoff matters or files are reused with specific people:

```text
01 Internal
02 Client Ready
03 Shared With Partner
04 Private Notes
```

Good for consulting, sales, interview prep, collaborative projects, and shared workspaces.

## Numbered Stable Areas

Use for large libraries that need stable navigation. Borrow the idea from Johnny.Decimal: numbered top-level areas make the map durable.

```text
00 Start Here
10-19 Brand Strategy
20-29 Customer And Shopper
30-39 Digital And Media
40-49 Measurement
90-99 Archive And Review
```

Use this when folder names are likely to grow and drift over time.

## Cleanup And Quarantine

Use review folders instead of deleting:

```text
90 Original Bundles
98 Needs Human Review
99 Cleanup Candidates
```

Put partial downloads, duplicate-looking files, orphan screenshots, old zips, and uncertain files here only after approval.

## Index Pattern

For archives and libraries, create a `00 Start Here - Index.md` that explains:
- What this folder is.
- Where to go for common needs.
- Which files are core references.
- Which areas are archival only.
- What still needs review.
