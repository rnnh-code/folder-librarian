# Folder Types

Use this reference to classify the folder before designing a structure. Mixed folders can have a primary type plus one or two secondary types.

## Active Work

Signals: drafts, current deliverables, client names, project plans, meeting notes, WIP exports, files modified recently.

Best structure: action-oriented. Use folders such as `01 Active Projects`, `02 Working Docs`, `03 References`, `04 Deliverables`, `90 Archive`.

Avoid: deep topic taxonomies that hide current next actions.

## Reference Library

Signals: playbooks, frameworks, PDFs, decks, training material, research, examples, old but useful professional material.

Best structure: topic/library-oriented. Use `00 Start Here`, core frameworks, topic areas, case studies, templates/tools, and archive/review areas.

Avoid: sorting by file type. A `PDFs` folder is tidy but rarely useful for retrieval.

## Admin Records

Signals: invoices, receipts, tax, legal, contracts, HR, healthcare, banking, immigration, insurance.

Best structure: time, source, and status. Use sortable dates like `YYYY-MM-DD`, clear `Final` versus `Draft`, and folder names that match how records will be retrieved later.

Avoid: renaming legal or financial source files aggressively if the original filename is evidence.

## Creative And Media

Signals: images, video, audio, design assets, fonts, exports, source project files, thumbnails.

Best structure: asset lifecycle. Use `Source`, `Working`, `Exports`, `Final`, `References`, and sometimes `By Campaign` or `By Platform`.

Avoid: mixing final deliverables with raw footage or generated exports.

## Data And Research

Signals: CSV, XLSX, JSON, transcripts, scraped data, survey exports, analysis notebooks, reports.

Best structure: provenance and reproducibility. Use `Raw`, `Processed`, `Analysis`, `Reports`, `Docs`, and `Archive`.

Avoid: editing raw data in place. Raw inputs should remain traceable.

## Backup Or Mixed Dump

Signals: many root-level files, duplicate names, old downloads, zips, screenshots, partial downloads, unclear provenance.

Best structure: staged triage. Use `00 Start Here`, `10 Likely Keep`, `20 Sort By Topic`, `90 Original Bundles`, `98 Needs Review`, `99 Cleanup Candidates`.

Avoid: destructive cleanup on the first pass. Backups often contain hidden evidence or context.

## Knowledge Vault

Signals: inbox/raw/wiki/index folders, source logs, topic maps, synthesized notes, compiled pages.

Best structure: preserve system contract. Treat live source folders as truth, refresh derived indexes after changes, and record processed and discarded sources.

Avoid: trusting old index files without checking live counts.
